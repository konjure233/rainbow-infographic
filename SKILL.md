# 彩虹渐变信息图生成器2.0

将文字内容智能提炼，生成柔和彩虹渐变风格的竖版信息图（HTML），并自动网页转图片，适合公众号、小红书、抖音等图文社交媒体分享。

## 核心产出

- 输出格式：单个 `.html` 文件（包含一页或多页卡片）
- 固定尺寸：每页 **1080px × 1440px**（3:4 比例），内部模块高度相加不得超过 1330px（防溢出）。
- 无论用户输入多长的文本，**必须且只能生成1页**。
- 强制信息压缩：提取最重要的核心概念和最多 3-6 个关键要点。放弃次要细节，确保视觉留白。
严禁输出 `` 这类多页结构，删除所有页脚的页码指示器。
- 保存到 `/home/workspace/outputs/` 目录

## 设计语言（六大核心特征）

这套设计语言是本 skill 的灵魂，每一张信息图都必须严格遵守：

### 1. 柔和彩虹渐变背景
整体背景使用极低饱和度的多色渐变（蓝→粉→黄→绿→蓝→紫），营造通透轻盈感。渐变方向为从上到下，色彩过渡自然柔和，不抢内容视线。

```css
background: linear-gradient(
  180deg,
  rgba(220, 230, 255, 0.6) 0%,
  rgba(255, 220, 230, 0.4) 20%,
  rgba(255, 240, 210, 0.4) 40%,
  rgba(210, 255, 220, 0.4) 60%,
  rgba(220, 230, 255, 0.5) 80%,
  rgba(240, 220, 255, 0.5) 100%
);
```

可根据文章主题微调色彩倾向（如科技主题偏蓝紫、金融主题偏金蓝），但始终保持"柔和彩虹"的基本调性。

### 2. 白色毛玻璃卡片分组
所有内容块用半透明白色卡片承载，带轻微模糊效果。

```css
.info-card {
  background: rgba(255,255,255,0.85);
  backdrop-filter: blur(10px);
  border-radius: 14px;
  padding: 26px 28px;
  border: 1px solid rgba(255,255,255,0.9);
}
```

卡片之间用网格布局排列（2列或3列），保持整齐对称。

### 3. 英文大写标签系统
每个内容模块上方放置灰色小字号、大字母间距的英文标签，增加国际感和信息层次。

```css
.card-tag {
  font-size: 12px;
  letter-spacing: 2px;
  color: #aaa;
  text-transform: uppercase;
}
```

英文标签应简洁概括该模块的主题，如 `BENCHMARK LEADER`、`TECHNICAL SPECS`、`INVESTMENT IMPLICATIONS`。

### 4. 彩色大号关键数字/关键词
核心数据或关键结论用 36px+ 加粗彩色文字，形成视觉焦点。每个模块的高亮颜色应有所不同，形成丰富但不杂乱的色彩节奏。

可用色板：
- 红色 `#e74c3c` — 强调、警示、核心数据
- 蓝色 `#4a6cf7` — 科技、信任、品牌
- 绿色 `#27ae60` — 增长、正面、环保
- 紫色 `#8e44ad` — 创新、高端
- 橙色 `#e67e22` — 活力、突破
- 青色 `#00b894` — 清新、健康

### 5. 背景水印装饰
底层放置超大半透明文字或数字（如关键指标 `1T`、`256K`、`340%`），增加画面丰富度但不干扰阅读。

```css
.bg-watermark {
  position: absolute;
  font-size: 200-280px;
  font-weight: 900;
  color: rgba(0,0,0,0.03);
  z-index: 0;
  font-family: 'Arial Black', sans-serif;
}
```

每页放置 2-3 个水印，错落分布在不同位置。水印内容从文章中提取最具视觉冲击力的数字或关键词。

### 6. 结构化信息层次
从上到下的信息流：
- **顶部标签**：来源/日期/类别（小字灰色）
- **大标题**：核心主题（48-52px 加粗，可带渐变色）
- **副标签**：英文辅助描述
- **核心概念横幅**：一句话总结全文核心（白色大卡片）
- **要点卡片组**：2-3列网格排列的信息模块
- **底部CTA/来源**：行动号召或数据来源声明

## 四阶段工作流

### 阶段一：内容分析与提炼

1. 阅读用户提供的文字，提取：
   - 主标题（1个，凝练有力）
   - 核心概念（1句话概括全文）
   - 关键要点（3-8个，每个含标题+简要描述）
   - 高亮数据（关键数字、百分比、里程碑）
   - 来源/作者/日期信息
2. 评估内容密度，决定页数：
   - ≤5个要点 → 单页
   - 6-10个要点 → 2页
   - 11+个要点 → 3页
3. 从内容中挑选 3-4 个适合做水印的数字/关键词

### 阶段二：结构规划

1. 确定每页包含哪些内容模块
2. 规划布局（哪些模块用2列、哪些用3列、哪些全宽）
3. 为每个模块分配英文标签和高亮颜色
4. 规划水印位置和内容

### 阶段三：HTML 生成

按照模板结构生成完整 HTML。生成前先读取 `assets/template.html` 获取完整的模板代码。

关键技术要求：
- 使用 Google Fonts 加载 Noto Sans SC
- 每页严格 1080×1440px，`overflow: hidden` 防溢出
- 所有卡片使用毛玻璃效果
- 网格布局用 CSS Grid
- 水印用绝对定位

### 阶段四：检查与输出

1. 检查每页内容是否在 1440px 内完整呈现
2. 检查色彩搭配是否和谐，高亮颜色是否足够丰富
3. 检查水印是否不干扰正文阅读
4. 检查英文标签是否准确
5. 写入 `/home/workspace/outputs/maghreb-rainbow-infographic.html`
6. 向用户呈现最终结果

## 多页拆分规则

- 每页必须在 1440px 高度内完整呈现，**严禁溢出**
- 宁可多分一页，也不压缩字号或间距
- 首页：顶部标签 + 大标题 + 核心概念 + 前几个要点
- 中间页：续接章节标题 + 后续要点模块
- 末页：最后的要点 + 结论/展望 + 来源信息
- 多页时每页右下角显示页码（如 `1 / 3`）
- 所有页共享统一的渐变背景和设计语言

### 防溢出容量估算（重要）

每页 1440px 减去上下 padding（约 110px）后，有效内容高度约 **1330px**。各组件的典型高度参考：

| 组件 | 估算高度 |
|------|----------|
| 顶部标签 + 主标题 + 副标签 | ~130px |
| concept-banner（含标题+1段文字） | ~140px |
| concept-banner（含标题+文字+2列子卡片） | ~260px |
| concept-banner（含标题+文字+3列子卡片） | ~260px |
| card-row-3（独立3列卡片） | ~180px |
| card-row-2（独立2列卡片，含大数字） | ~200px |
| bordered-left 趋势列表（5条） | ~220px |
| footer-note | ~50px |

**经验法则：每页最多放 4-5 个 concept-banner 模块。** 如果一页有 5 个以上模块，必须通过以下方式之一确保不溢出：
1. 缩小卡片内 padding（从 26px 降到 18-20px）
2. 缩小模块间距（从 22px 降到 14-16px）
3. 缩小正文字号（从 17px 降到 15px）
4. 精简文字内容
5. 拆分到下一页

## 内容适配指南

不同类型的文章，信息图的结构有所侧重：

**科技产品发布类**（如 Kimi K2-Thinking）
- 核心概念横幅突出产品定位
- 基准测试数据用2列卡片 + 彩色大数字
- 技术参数用列表型卡片
- 底部放体验入口CTA

**数据分析/商业洞察类**（如 AI科技股分析）
- 按主题分大模块，每个模块含标题横幅 + 子卡片
- 重点数据用彩色高亮
- 底部放结论/展望 + 数据来源声明

**教程/方法论类**
- 步骤编号清晰
- 每步一个卡片模块
- 关键技巧用彩色高亮

**行业报告/趋势类**
- 按趋势/发现分组
- 大量使用数据高亮
- 底部放启示和来源
